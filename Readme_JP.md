# MulMollaa
Ollama / llama-server と連携し、複数の LLM が相互に会話できるローカルアプリケーションです。  
会話の流れはすべてプロンプトで制御でき、ユニット（会議室）の概念で複数エージェントを管理できます。

> ※ Python 版はクローズドソースのため非公開です。  
> ※ 公開されているのは EXE 版のみです。

---

## 特徴
- 複数の LLM が無制限に会話可能  
- 会話の制御はすべてプロンプトで行える  
- Web UI で操作  
- Ollama / llama-server の API を利用  
- 初回起動時にユーザーデータを自動生成  
- EXE 1 ファイルで動作（Nuitka onefile）

---

## 動作環境
- **OS**：Windows 11  
- **必要なもの**：  
  - Ollama または llama-server がローカルで起動していること  
  - モデルがロードされていること  
- **Python 版**：クローズドソースのため非公開

---

## 使い方

### 1. 事前準備
MulMollaa を起動する前に、以下のいずれかをローカルで起動しておいてください：

- **Ollama**
- **llama-server**
- または両方

API が有効な状態である必要があります。

---

### 2. 起動
EXE をダブルクリックすると以下が自動で行われます：

- コンソールの起動  
- ブラウザの起動  
- TEMP フォルダへのファイル展開  
- ユーザープロファイルに `.mulmollaa` フォルダを作成  
  - 設定ファイル  
  - プロンプトファイル  

---

## AI チャットの使い方
設定した LLM の名前の前に **半角の @** を付けて呼び出します。

例：@main 徳川家康は最初から天下を見据えていたと思う？

※ 名前の後ろには半角スペースを入れてください。

---

## LLM 設定項目

name              : 任意の名称
node              : Ollama または llama
history-limit     : 何個前まで記憶するか
connection        : API 接続用 URL
loadmodels-model  : Ollama 用モデル指定
modelfile         : llama 用モデル指定
vision            : まだ未対応
unitname          : 会議室のような概念
prompt            : システムプロンプト

### 例：API 接続用 URL

llama   http://localhost:8080/v1/chat/completions
Ollama  http://localhost:11434/api/generate


---

## ユニット設定
ユニットは「会議室」のような概念です。  
現在は主に以下を使用します：

- `unitname`（ユニット名）
- `order`（表示順）

その他の項目は今後機能追加予定です。

## 今後の開発予定（Roadmap）

MulMollaa は現在も開発中であり、以下の機能を順次追加予定です。

- **共通システムプロンプト**
  - ユニット内LLMの役割設定共通化
  - プロンプト入力補助

- **LLMの有効・無効設定**

- **ユニットベースのワークフロー UI**
  - ユニット同士の接続を視覚的に編集できる機能、ルーティング

- **Function Calling 対応**
  - LLM による外部関数呼び出しのサポート、Skill

- **Vision（画像入力）対応**
  - 画像を LLM に渡して解析する機能

- **自律学習ループ 対応**
  - 入出力・生成情報の自律学習

- **UI のアップグレード**
  - チャット UI の改善
  - ダークモード対応
  - レイアウト調整

- **ログ管理機能**
  - 会話ログの保存・読み込み

※ 上記は予定であり、順番や内容は変更される可能性があります。
